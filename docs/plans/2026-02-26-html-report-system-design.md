# HTML报告系统设计文档

**日期**: 2026-02-26
**版本**: 1.0
**状态**: 已批准

## 项目概述

将现有的PDF报告生成系统扩展为HTML静态网页报告系统，支持在线查看和静态文件下载的混合模式，提供现代化的用户体验。

## 设计目标

1. **信息完整性**: 完整保留PDF模板中的所有信息项
2. **访问灵活性**: 同时支持在线查看和静态HTML下载
3. **现代化体验**: 响应式设计、交互功能和数据可视化
4. **系统兼容性**: 最小化对现有架构的改动
5. **美观大方**: 现代网页布局，优于传统PDF展示

## 技术架构

### 核心组件结构

```
现有系统 + 新增组件
├── modules/report_generator/
│   ├── generator.py (扩展 - 添加HTML生成方法)
│   └── html_generator.py (新增 - HTML专用生成器)
├── templates/
│   ├── report.html (保留 - PDF版本)
│   ├── report_web.html (新增 - 在线版本)
│   └── report_static.html (新增 - 静态版本)
├── static/ (新增)
│   ├── css/report.css (报告样式)
│   ├── js/report.js (交互功能)
│   └── images/ (图标资源)
└── api/routers/
    └── reports.py (新增 - 报告访问API)
```

### API端点设计

**新增路由**:
- `GET /api/v1/reports/{task_id}/view` - 在线查看HTML报告
- `GET /api/v1/reports/{task_id}/download` - 下载静态HTML文件
- `GET /api/v1/reports/{task_id}/data` - 获取报告JSON数据

### 数据流设计

1. **报告生成阶段**:
   - 保持现有Celery任务结构
   - 扩展ReportGenerator类，添加HTML生成能力
   - 同时生成PDF（向后兼容）和HTML数据
   - HTML数据存储路径: `reports/{task_id}/report.html`

2. **报告访问阶段**:
   - 在线查看: 从MinIO获取数据，使用轻量级模板渲染
   - 静态下载: 生成自包含HTML文件，内联所有资源

## 模板系统设计

### 双模板策略

**在线版本** (`report_web.html`):
- 引用外部CSS/JS资源
- 响应式设计，支持移动端
- 轻量级，快速加载
- 支持交互功能（折叠展开、数据筛选）

**静态版本** (`report_static.html`):
- 内联所有CSS/JS资源
- 包含完整打印样式
- 自包含，无外部依赖
- 优化离线查看体验

### 信息架构设计

基于PDF模板，HTML版本包含以下完整信息区域：

1. **封面区域**
   - 现代卡片式设计
   - 突出显示应用名称和风险等级
   - 生成时间和任务信息

2. **基本信息面板**
   - 网格布局展示应用元数据
   - 包名、版本、文件大小、哈希值
   - SDK版本信息

3. **可调证线索信息**
   - 嫌疑域名信息（置信度可视化）
   - 网络行为分析表格
   - 加固信息检测
   - 嫌疑开发者信息（电话号码等）

4. **附件信息**
   - 证书信息详细展示
   - 应用权限分析（风险等级标色）
   - 静态分析信息（域名、IP列表）
   - 动态分析信息（DNS查询、访问域名）

5. **截图画廊**
   - 响应式图片展示
   - 支持放大查看（灯箱效果）
   - 按阶段分组展示

6. **风险评估仪表板**
   - 直观的风险评分展示
   - 风险因素列表
   - 分析总结

## 技术实现细节

### HTML生成器类设计

```python
class HTMLReportGenerator:
    def __init__(self, template_dir: str = "templates"):
        self.template_dir = template_dir
        self.env = Environment(...)

    def generate_web_report(self, data: Dict) -> str:
        """生成在线查看的HTML"""

    def generate_static_report(self, data: Dict) -> str:
        """生成静态下载的HTML（内联资源）"""

    def inline_resources(self, html: str) -> str:
        """内联CSS/JS资源到HTML中"""
```

### 资源处理策略

**在线版本**:
- CSS/JS文件通过FastAPI静态文件服务
- 图片资源从MinIO获取
- 支持CDN加速

**静态版本**:
- 所有CSS内联到`<style>`标签
- 所有JS内联到`<script>`标签
- 图片转换为base64内联
- 生成完全自包含的HTML文件

## 兼容性考虑

1. **向后兼容**: 保留现有PDF生成功能
2. **数据兼容**: 使用相同的数据处理逻辑
3. **API兼容**: 现有API端点保持不变
4. **存储兼容**: HTML文件存储到MinIO，不影响现有结构

## 性能优化

1. **缓存策略**: 静态资源浏览器缓存
2. **压缩优化**: HTML/CSS/JS压缩
3. **图片优化**: 截图压缩和懒加载
4. **响应式加载**: 按需加载大型数据表格

## 安全考虑

1. **XSS防护**: 模板自动转义用户数据
2. **访问控制**: 报告访问需要有效的task_id
3. **资源隔离**: 静态资源独立域名服务
4. **数据验证**: 输入数据严格验证

## 部署策略

1. **渐进式部署**: 先部署HTML功能，保留PDF作为备选
2. **A/B测试**: 支持用户选择报告格式
3. **监控指标**: 报告生成成功率、访问性能
4. **回滚计划**: 快速回退到纯PDF模式
